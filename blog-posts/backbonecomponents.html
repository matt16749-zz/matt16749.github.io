Backbone Components

Backbone.Router

Backbone's Router component watches for any changes to URL router and provides a list of callback functions that grab JSON data and renders a view.

Backbone.View

Paired with an HTML template that can be populated with model data via Backbone.Router. The actual functions that populate the data come from the view.

Backbone.Collection/Model

A collection consists of multiple models. Models take data JSON endpoints of an API(which would be our Rails)

Backbone Workflow

  1. Instantiate Backbone router to listen for URL hashchange events.
  2. On specific hashchange, Router will fetch model/collection data from server.
  3. After fetching model/collection data, Router instantiates a parent collection view and render into the DOM.
  4. Parent collection view also should have a reset function so that when server lists new data, the collection view will automatically iterate through each model data and output the subset model view within the collection view.
